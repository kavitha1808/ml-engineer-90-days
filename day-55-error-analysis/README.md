# Day 55 – Error Analysis & Debugging ML Models

## 📌 Objective
This project focuses on understanding how to analyze and debug machine learning models by identifying errors and improving performance.

---

## 📊 Dataset
- Iris Dataset (from sklearn)
- Multi-class classification problem

---

## ⚙️ Workflow

1. Load dataset
2. Split into train and test sets
3. Train Logistic Regression model
4. Evaluate model performance
5. Perform error analysis on misclassified samples

---

## 📈 Evaluation Metrics Used

- Accuracy Score
- Confusion Matrix
- Classification Report (Precision, Recall, F1-score)

---

## 🔍 Error Analysis Steps

- Identified misclassified samples
- Compared actual vs predicted labels
- Analyzed feature distributions of errors
- Checked class imbalance

---

## 🧠 Key Learnings

- Error analysis is essential for improving ML models
- Confusion matrix helps understand misclassification patterns
- Data quality impacts model performance more than algorithms
- Debugging ML is a structured process, not trial and error

---

## 🛠️ Tech Stack

- Python
- Pandas
- NumPy
- Scikit-learn
